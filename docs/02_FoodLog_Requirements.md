# FoodLog 요구사항 정의서

## 1. 문서 개요

| 항목 | 내용 |
| --- | --- |
| 문서명 | FoodLog 요구사항 정의서 |
| 작성 목적 | FoodLog MVP의 기능, 화면, 구현 범위를 명확히 정의하기 위함 |
| 대상 프로젝트 | FoodLog |
| 문서 기준 | FoodLog PRD 및 현재 MVP 기능 범위 |
| 프로젝트 단계 | 프로젝트 초기 기획 및 화면 설계 단계 |

본 문서는 FoodLog 프로젝트에서 구현할 주요 기능과 화면 요구사항을 정리한 문서이다.  
FoodLog는 사용자가 먹은 음식을 기록하고, 음식 히스토리를 확인하며, 간단한 식단 피드백을 받을 수 있는 음식 기록 서비스이다.

## 2. 프로젝트 개요

| 항목 | 내용 |
| --- | --- |
| 프로젝트명 | FoodLog |
| 서비스 목적 | 사용자의 음식 기록 관리와 간단한 식단 피드백 제공 |
| 주요 사용자 | 식단 기록을 간단히 관리하고 싶은 사용자 |
| 주요 기능 | 음식 기록 추가, 조회, 수정, 삭제 및 식단 피드백 확인 |
| 백엔드 기술 | Node.js, Express |
| 데이터 저장 방식 | DB 없이 서버 메모리의 임시 배열 사용 |

FoodLog는 음식 기록을 추가, 조회, 수정, 삭제할 수 있는 기본 기능을 중심으로 구성한다.  
현재 음식 데이터는 고유 번호, 음식 이름, 카테고리, 칼로리, 메모, 기록 일시 항목으로 관리한다.

## 4. 비기능 요구사항

| 요구사항 | 설명 | 우선순위 |
| --- | --- | --- |
| 사용하기 쉬운 화면 | 사용자가 목록, 추가, 상세, 피드백 화면을 쉽게 이동할 수 있어야 한다. | 높음 |

| 가벼운 프로젝트 구조 | MVP 범위에 맞게 복잡한 인증, DB, 배포 구조는 제외한다. | 보통 |

## 5. 화면 요구사항

## 6. MVP 범위

| 기능 | 포함 여부 | 설명 |
| --- | --- | --- |
| 음식 기록 추가 | 포함 | 사용자가 음식 기록을 새로 등록한다. |
| 음식 기록 목록 조회 | 포함 | 전체 음식 기록을 목록으로 확인한다. |
| 음식 기록 상세 조회 | 포함 | 특정 음식 기록의 상세 정보를 확인한다. |
| 음식 기록 수정 | 포함 | 기존 음식 기록을 수정한다. |
| 음식 기록 삭제 | 포함 | 불필요한 음식 기록을 삭제한다. |
| 식단 피드백 확인 | 포함 | 규칙 기반으로 간단한 피드백을 제공한다. |
| 주요 화면 설계 | 포함 | 목록, 추가, 상세/수정, 피드백 화면을 설계한다. |

## 7. 제외 범위

| 제외 항목 | 설명 |
| --- | --- |
| 로그인/회원가입 | MVP 단계에서는 사용자 인증 기능을 구현하지 않는다. |
| 데이터베이스 저장 | 현재는 DB 없이 서버 메모리의 임시 배열로 데이터를 관리한다. |
| 실제 AI API 연동 | AI API는 향후 확장 방향으로만 정의하고 MVP에는 포함하지 않는다. |
| 이미지 업로드 | 음식 사진 등록 기능은 MVP 범위에서 제외한다. |
| 복잡한 통계 기능 | 일별, 주별, 월별 통계 기능은 향후 확장 기능으로 둔다. |
