# FoodLog 와이어프레임

## 1. 문서 개요

| 항목 | 내용 |
| --- | --- |
| 문서명 | FoodLog 와이어프레임 |
| 작성 목적 | FoodLog의 주요 화면 구조와 사용자 흐름을 간단히 시각화하기 위함 |
| 기준 문서 | FoodLog PRD, 요구사항 정의서, 사용자 시나리오 |
| 화면 기준 | 모바일 화면 기준 |
| 프로젝트 단계 | 프로젝트 초기 기획 및 화면 설계 단계 |

본 문서는 FoodLog 웹 서비스의 주요 화면을 모바일 우선 관점에서 와이어프레임 형태로 정리한 문서이다.  
FoodLog는 PC와 모바일 브라우저에서 사용할 수 있으며, 사용자가 먹은 음식을 기록하고, 음식 히스토리를 확인하며, 간단한 식단 피드백을 받을 수 있는 음식 기록 서비스이다.  
초기 화면 설계는 모바일 사용성을 우선하여 모바일 화면 기준으로 구성한다.

## 2. 전체 화면 구성

FoodLog의 MVP 화면은 다음 4개 화면으로 구성한다.

| 번호 | 화면명 | 주요 목적 |
| --- | --- | --- |
| 1 | 음식 기록 목록 화면 | 등록된 음식 기록을 목록으로 확인 |
| 2 | 음식 기록 추가 화면 | 새로운 음식 기록 입력 및 저장 |
| 3 | 음식 기록 상세/수정 화면 | 선택한 기록의 상세 확인, 수정, 삭제 |
| 4 | 식단 피드백 화면 | 기록된 음식 데이터를 바탕으로 간단한 피드백 확인 |

## 3. 음식 기록 목록 화면

### 화면 목적

사용자가 등록한 음식 기록을 한눈에 확인하는 메인 화면이다.  
음식 기록 추가, 상세 확인, 식단 피드백 화면으로 이동하는 출발점 역할을 한다.

### 주요 구성 요소

| 구성 요소 | 설명 |
| --- | --- |
| 상단 제목 | 서비스명 FoodLog 표시 |
| 화면 제목 | 오늘의 음식 기록 또는 음식 기록 목록 표시 |
| 음식 추가 버튼 | 음식 기록 추가 화면으로 이동 |
| 피드백 보기 버튼 | 식단 피드백 화면으로 이동 |
| 음식 기록 카드 | 음식 이름, 카테고리, 칼로리, 메모 요약 표시 |

### 사용자 행동

| 사용자 행동 | 결과 |
| --- | --- |
| 음식 추가 버튼 선택 | 음식 기록 추가 화면으로 이동 |
| 피드백 보기 버튼 선택 | 식단 피드백 화면으로 이동 |
| 음식 기록 카드 선택 | 음식 기록 상세/수정 화면으로 이동 |

### 화면 이동 흐름

| 이동 전 | 사용자 행동 | 이동 후 |
| --- | --- | --- |
| 음식 기록 목록 화면 | 음식 추가 선택 | 음식 기록 추가 화면 |
| 음식 기록 목록 화면 | 기록 카드 선택 | 음식 기록 상세/수정 화면 |
| 음식 기록 목록 화면 | 피드백 보기 선택 | 식단 피드백 화면 |

### 모바일 화면 와이어프레임

```text
+--------------------------------+
| FoodLog                        |
+--------------------------------+
| 오늘의 음식 기록               |
| [ + 음식 추가 ] [ 피드백 보기 ]|
+--------------------------------+
| 사과                           |
| 과일 / 95 kcal                 |
| 오후 간식으로 먹음             |
+--------------------------------+
| 밥                             |
| 곡류 / 206 kcal                |
| 점심으로 먹음                  |
+--------------------------------+
| 달걀                           |
| 단백질 / 78 kcal               |
| 아침으로 먹음                  |
+--------------------------------+
```

## 4. 음식 기록 추가 화면

### 화면 목적

사용자가 먹은 음식을 새 기록으로 입력하고 저장하는 화면이다.  
입력 항목은 음식 이름, 카테고리, 칼로리, 메모로 단순하게 구성한다.

### 주요 구성 요소

| 구성 요소 | 설명 |
| --- | --- |
| 상단 제목 | FoodLog 표시 |
| 화면 제목 | 음식 기록 추가 표시 |
| 음식 이름 입력란 | 사용자가 먹은 음식 이름 입력 |
| 카테고리 입력란 | 음식 종류 또는 분류 입력 |
| 칼로리 입력란 | 음식 칼로리 입력 |
| 메모 입력란 | 간단한 메모 입력 |
| 저장 버튼 | 입력한 음식 정보를 새 기록으로 저장 |
| 취소 버튼 | 입력을 취소하고 목록 화면으로 이동 |

### 사용자 행동

| 사용자 행동 | 결과 |
| --- | --- |
| 음식 정보를 입력한다 | 입력값이 화면에 반영된다. |
| 저장 버튼을 선택한다 | 새 음식 기록이 저장되고 목록 화면으로 이동한다. |
| 취소 버튼을 선택한다 | 입력을 취소하고 목록 화면으로 돌아간다. |

### 화면 이동 흐름

| 이동 전 | 사용자 행동 | 이동 후 |
| --- | --- | --- |
| 음식 기록 추가 화면 | 저장 선택 | 음식 기록 목록 화면 |
| 음식 기록 추가 화면 | 취소 선택 | 음식 기록 목록 화면 |

### 모바일 화면 와이어프레임

```text
+--------------------------------+
| FoodLog                        |
+--------------------------------+
| 음식 기록 추가                 |
+--------------------------------+
| 음식 이름                      |
| [ 사과                     ]   |
+--------------------------------+
| 카테고리                       |
| [ 과일                     ]   |
+--------------------------------+
| 칼로리                         |
| [ 95                       ]   |
+--------------------------------+
| 메모                           |
| [ 오후 간식으로 먹음      ]    |
+--------------------------------+
| [ 저장하기 ]   [ 취소 ]        |
+--------------------------------+
```

## 5. 음식 기록 상세/수정 화면

### 화면 목적

사용자가 선택한 음식 기록의 상세 정보를 확인하고, 필요한 경우 내용을 수정하거나 삭제하는 화면이다.

### 주요 구성 요소

| 구성 요소 | 설명 |
| --- | --- |
| 상단 제목 | FoodLog 표시 |
| 화면 제목 | 음식 기록 상세 표시 |
| 음식 이름 입력란 | 선택한 음식 이름 표시 및 수정 |
| 카테고리 입력란 | 선택한 음식 카테고리 표시 및 수정 |
| 칼로리 입력란 | 선택한 음식 칼로리 표시 및 수정 |
| 메모 입력란 | 선택한 음식 메모 표시 및 수정 |
| 기록 일시 | 음식 기록 일시 표시 |
| 저장 버튼 | 수정한 내용을 저장 |
| 삭제 버튼 | 선택한 음식 기록 삭제 |
| 목록 버튼 | 음식 기록 목록 화면으로 이동 |

### 사용자 행동

| 사용자 행동 | 결과 |
| --- | --- |
| 기록 내용을 확인한다 | 선택한 음식 기록의 상세 정보를 확인한다. |
| 입력값을 수정하고 저장한다 | 수정된 내용이 반영된다. |
| 삭제 버튼을 선택한다 | 해당 음식 기록이 삭제되고 목록 화면으로 이동한다. |
| 목록 버튼을 선택한다 | 목록 화면으로 돌아간다. |

### 화면 이동 흐름

| 이동 전 | 사용자 행동 | 이동 후 |
| --- | --- | --- |
| 음식 기록 상세/수정 화면 | 저장 선택 | 음식 기록 상세/수정 화면 또는 목록 화면 |
| 음식 기록 상세/수정 화면 | 삭제 선택 | 음식 기록 목록 화면 |
| 음식 기록 상세/수정 화면 | 목록 선택 | 음식 기록 목록 화면 |

### 모바일 화면 와이어프레임

```text
+--------------------------------+
| FoodLog                        |
+--------------------------------+
| 음식 기록 상세                 |
+--------------------------------+
| 음식 이름                      |
| [ 사과                     ]   |
+--------------------------------+
| 카테고리                       |
| [ 과일                     ]   |
+--------------------------------+
| 칼로리                         |
| [ 95                       ]   |
+--------------------------------+
| 메모                           |
| [ 오후 간식으로 먹음      ]    |
+--------------------------------+
| 기록 일시                      |
| 오전 11:09                     |
+--------------------------------+
| [ 저장하기 ] [ 삭제 ] [ 목록 ] |
+--------------------------------+
```

## 6. 식단 피드백 화면

### 화면 목적

사용자가 기록한 음식 데이터를 바탕으로 간단한 식단 피드백을 확인하는 화면이다.  
현재 MVP에서는 AI 분석이 아닌 규칙 기반 피드백을 제공한다.

### 주요 구성 요소

| 구성 요소 | 설명 |
| --- | --- |
| 상단 제목 | FoodLog 표시 |
| 화면 제목 | 식단 피드백 표시 |
| 총 기록 수 | 등록된 음식 기록 개수 표시 |
| 총 칼로리 | 기록된 음식의 칼로리 합계 표시 |
| 카테고리 요약 | 음식 카테고리별 기록 현황 표시 |
| 피드백 메시지 | 규칙 기반 식단 피드백 표시 |
| 목록 버튼 | 음식 기록 목록 화면으로 이동 |

### 사용자 행동

| 사용자 행동 | 결과 |
| --- | --- |
| 피드백 화면을 확인한다 | 음식 기록 요약과 피드백 메시지를 확인한다. |
| 목록 버튼을 선택한다 | 음식 기록 목록 화면으로 돌아간다. |

### 화면 이동 흐름

| 이동 전 | 사용자 행동 | 이동 후 |
| --- | --- | --- |
| 식단 피드백 화면 | 목록 선택 | 음식 기록 목록 화면 |

### 모바일 화면 와이어프레임

```text
+--------------------------------+
| FoodLog                        |
+--------------------------------+
| 식단 피드백                    |
+--------------------------------+
| 총 기록 수                     |
| 3개                            |
+--------------------------------+
| 총 칼로리                      |
| 379 kcal                       |
+--------------------------------+
| 카테고리 요약                  |
| 과일: 1 / 곡류: 1 / 단백질: 1  |
+--------------------------------+
| 피드백                         |
| 다양한 카테고리의 음식이       |
| 기록되어 있습니다.             |
| 꾸준히 식사 후 기록해보세요.   |
+--------------------------------+
| [ 목록으로 ]                   |
+--------------------------------+
```

## 7. 전체 화면 이동 요약

| 시작 화면 | 사용자 행동 | 이동 화면 |
| --- | --- | --- |
| 음식 기록 목록 화면 | 음식 추가 선택 | 음식 기록 추가 화면 |
| 음식 기록 추가 화면 | 저장 또는 취소 | 음식 기록 목록 화면 |
| 음식 기록 목록 화면 | 음식 기록 선택 | 음식 기록 상세/수정 화면 |
| 음식 기록 상세/수정 화면 | 목록 또는 삭제 선택 | 음식 기록 목록 화면 |
| 음식 기록 목록 화면 | 피드백 보기 선택 | 식단 피드백 화면 |
| 식단 피드백 화면 | 목록 선택 | 음식 기록 목록 화면 |

## 8. 한 줄 요약

FoodLog는 PC와 모바일 브라우저에서 사용할 수 있는 웹 서비스이며, 초기 화면 설계는 모바일 사용성을 우선하여 음식 기록 목록을 중심으로 추가, 상세/수정, 피드백 화면으로 이동하는 단순한 구조로 설계한다.
