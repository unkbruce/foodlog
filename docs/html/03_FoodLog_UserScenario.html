User Scenario

FoodLog 사용자 시나리오

FoodLog 사용자가 음식 기록을 추가하고, 히스토리를 확인하며, 수정과 삭제 및 피드백 확인까지 이어지는 주요 흐름을 정리한 문서입니다.

프로젝트명FoodLog
기준 문서PRD, 요구사항 정의서
핵심 흐름기록, 확인, 수정, 피드백

PDF 저장 시 브라우저 인쇄 설정에서 머리글과 바닥글을 끄면 문서 본문만 깔끔하게 출력됩니다.

핵심 요약 사용자는 목록 화면에서 음식 기록을 추가하고, 목록과 상세 화면에서 기록을 확인한다. 필요한 경우 수정 또는 삭제를 수행하며, 피드백 화면에서 기록 개수, 총 칼로리, 카테고리 분포를 기반으로 한 안내를 확인한다.

1. 문서 개요

항목내용
문서명FoodLog 사용자 시나리오
작성 목적FoodLog 서비스의 주요 사용자 행동 흐름과 시스템 반응을 정리하기 위함
기준 문서FoodLog PRD, FoodLog 요구사항 정의서
프로젝트 단계프로젝트 초기 기획 및 화면 설계 단계

본 문서는 FoodLog 사용자가 서비스를 이용하는 주요 흐름을 시나리오 형태로 정리한 문서이다. FoodLog는 사용자가 먹은 음식을 기록하고, 음식 히스토리를 확인하며, 간단한 식단 피드백을 받을 수 있는 음식 기록 서비스이다.

2. 대표 사용자 설정

대표 사용자설명주요 니즈
식단 기록을 처음 시작하는 사용자식단 관리를 해보고 싶지만 복잡한 앱 사용에는 익숙하지 않은 사용자간단한 입력, 쉬운 목록 확인, 기본적인 피드백
하루에 먹은 음식을 간단히 관리하고 싶은 사용자하루 동안 먹은 음식을 기록하고 나중에 확인하고 싶은 사용자음식 히스토리 확인, 칼로리 참고, 기록 수정 및 삭제

FoodLog의 대표 사용자는 전문적인 식단 관리보다는 간단한 기록과 확인을 원하는 사용자로 설정한다.

3. 핵심 사용자 시나리오

3.1 음식 기록 추가 시나리오

항목내용
시작 조건사용자가 음식 기록 목록 화면에 접속해 있다.
사용자 목표먹은 음식을 새 기록으로 저장한다.
주요 입력값음식 이름, 카테고리, 칼로리, 메모
기대 결과새 음식 기록이 생성되고 목록에 표시된다.

3.2 음식 기록 목록 조회 시나리오

항목내용
시작 조건사용자가 FoodLog에 접속한다.
사용자 목표지금까지 기록한 음식 히스토리를 확인한다.
표시 정보음식 이름, 카테고리, 칼로리, 기록 일시
기대 결과등록된 음식 기록이 목록 형태로 표시된다.

3.3 음식 기록 상세 조회 시나리오

항목내용
시작 조건음식 기록 목록에 하나 이상의 기록이 있다.
사용자 목표선택한 음식 기록의 자세한 내용을 확인한다.
표시 정보고유 번호, 음식 이름, 카테고리, 칼로리, 메모, 기록 일시
기대 결과선택한 음식 기록의 상세 정보가 표시된다.

3.4 음식 기록 수정 시나리오

항목내용
시작 조건사용자가 음식 기록 상세/수정 화면에 접속해 있다.
사용자 목표기존 음식 기록의 내용을 수정한다.
수정 가능 항목음식 이름, 카테고리, 칼로리, 메모
기대 결과수정된 내용이 저장되고 목록 또는 상세 화면에 반영된다.

3.5 음식 기록 삭제 시나리오

항목내용
시작 조건사용자가 음식 기록 상세/수정 화면에 접속해 있다.
사용자 목표불필요하거나 잘못 등록된 기록을 삭제한다.
처리 대상선택한 음식 기록
기대 결과해당 음식 기록이 목록에서 제거된다.

3.6 식단 피드백 확인 시나리오

항목내용
시작 조건음식 기록이 저장되어 있다.
사용자 목표자신의 식단 상태를 간단히 참고한다.
기준 데이터기록 개수, 총 칼로리, 카테고리 분포
기대 결과규칙 기반 피드백 메시지가 표시된다.

4. 사용자 여정 흐름

순서단계사용자 행동시스템 역할
1목록 화면 접속사용자가 FoodLog에 접속한다.음식 기록 목록을 보여준다.
2음식 추가사용자가 음식 추가 버튼을 누르고 정보를 입력한다.입력된 정보를 새 음식 기록으로 저장한다.
3기록 확인사용자가 목록에서 음식 기록을 확인한다.저장된 음식 기록을 리스트로 표시한다.
4수정 또는 삭제사용자가 특정 기록을 선택해 수정하거나 삭제한다.선택한 기록을 수정하거나 삭제한다.
5피드백 확인사용자가 식단 피드백 화면으로 이동한다.기록 데이터를 기준으로 간단한 피드백을 제공한다.

5. 시나리오별 상세 표

5.1 음식 기록 추가

단계사용자 행동시스템 반응기대 결과
1음식 기록 목록 화면에서 추가 버튼을 선택한다.음식 기록 추가 화면을 표시한다.사용자가 새 기록을 입력할 수 있다.
2음식 이름, 카테고리, 칼로리, 메모를 입력한다.입력값을 화면에 반영한다.등록할 음식 정보가 준비된다.
3등록 버튼을 선택한다.입력값을 확인하고 새 음식 기록을 생성한다.새 기록이 저장된다.
4목록 화면으로 이동한다.생성된 기록을 목록에 표시한다.사용자가 추가된 기록을 확인할 수 있다.

5.2 음식 기록 목록 조회

단계사용자 행동시스템 반응기대 결과
1FoodLog에 접속한다.저장된 음식 기록 목록을 불러온다.사용자가 기록한 음식 히스토리를 확인할 수 있다.
2목록 화면을 확인한다.음식 이름, 카테고리, 칼로리, 기록 일시를 표시한다.사용자가 음식 히스토리를 확인할 수 있다.
3특정 기록을 선택한다.선택한 기록의 상세 화면으로 이동한다.개별 음식 기록을 확인할 수 있다.

5.3 음식 기록 상세 조회

단계사용자 행동시스템 반응기대 결과
1목록에서 특정 음식 기록을 선택한다.선택한 음식 기록의 상세 정보를 불러온다.사용자가 선택한 기록의 내용을 확인할 수 있다.
2상세 화면을 확인한다.음식 이름, 카테고리, 칼로리, 메모, 기록 일시를 표시한다.사용자가 상세 내용을 확인할 수 있다.
3뒤로가기 버튼을 선택한다.목록 화면으로 이동한다.사용자가 다시 전체 기록을 확인할 수 있다.

5.4 음식 기록 수정

단계사용자 행동시스템 반응기대 결과
1상세/수정 화면에서 수정할 값을 입력한다.변경된 입력값을 화면에 반영한다.수정할 내용이 준비된다.
2저장 버튼을 선택한다.선택한 음식 기록의 수정 내용을 저장한다.음식 기록이 수정된다.
3수정된 내용을 확인한다.수정된 데이터를 화면에 표시한다.사용자가 수정 결과를 확인할 수 있다.

5.5 음식 기록 삭제

단계사용자 행동시스템 반응기대 결과
1상세/수정 화면에서 삭제 버튼을 선택한다.삭제 요청을 처리한다.선택한 기록이 삭제 대상이 된다.
2시스템이 선택한 음식 기록을 삭제한다.해당 음식 기록을 목록에서 제거한다.삭제된 기록이 더 이상 표시되지 않는다.
3목록 화면으로 이동한다.삭제된 기록을 제외한 목록을 표시한다.사용자가 삭제 결과를 확인할 수 있다.

5.6 식단 피드백 확인

단계사용자 행동시스템 반응기대 결과
1목록 화면에서 피드백 화면으로 이동한다.음식 기록 요약 정보를 표시한다.사용자가 피드백 화면을 확인할 수 있다.
2사용자가 화면의 요약 정보를 확인한다.기록 개수, 총 칼로리, 카테고리 요약을 표시한다.사용자가 음식 기록 현황을 확인할 수 있다.
3사용자가 피드백 메시지를 확인한다.기록 데이터를 기준으로 적절한 피드백을 제공한다.사용자가 자신의 식단 상태를 간단히 참고할 수 있다.

6. 예외 상황

예외 상황사용자 행동시스템 반응기대 결과
입력값이 비어 있는 경우음식 이름, 카테고리, 칼로리 중 일부를 입력하지 않는다.필수 입력값이 필요하다는 메시지를 표시한다.사용자가 빠진 값을 입력한 뒤 다시 등록할 수 있다.
칼로리에 숫자가 아닌 값을 입력한 경우칼로리 입력란에 문자나 잘못된 값을 입력한다.숫자 입력이 필요하다는 안내를 표시한다.올바른 숫자 값으로 수정하도록 유도한다.
삭제하려는 기록이 없는 경우이미 삭제되었거나 존재하지 않는 기록을 삭제하려고 한다.기록을 찾을 수 없다는 메시지를 표시한다.사용자가 현재 목록 기준으로 다시 확인할 수 있다.

7. 한 줄 요약

요약 FoodLog는 사용자가 먹은 음식을 간단히 기록하고, 기록된 히스토리를 확인하며, 기본적인 식단 피드백까지 확인할 수 있는 음식 기록 서비스이다.
