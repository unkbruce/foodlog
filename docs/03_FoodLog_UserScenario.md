# FoodLog 사용자 시나리오

## 1. 문서 개요

| 항목 | 내용 |
| --- | --- |
| 문서명 | FoodLog 사용자 시나리오 |
| 작성 목적 | FoodLog 서비스의 주요 사용자 행동 흐름과 시스템 반응을 정리하기 위함 |
| 기준 문서 | FoodLog PRD, FoodLog 요구사항 정의서 |
| 프로젝트 단계 | 프로젝트 초기 기획 및 화면 설계 단계 |

본 문서는 FoodLog 사용자가 서비스를 이용하는 주요 흐름을 시나리오 형태로 정리한 문서이다.  
FoodLog는 사용자가 먹은 음식을 기록하고, 음식 히스토리를 확인하며, 간단한 식단 피드백을 받을 수 있는 음식 기록 서비스이다.

## 2. 대표 사용자 설정

| 대표 사용자 | 설명 | 주요 니즈 |
| --- | --- | --- |
| 식단 기록을 처음 시작하는 사용자 | 식단 관리를 해보고 싶지만 복잡한 앱 사용에는 익숙하지 않은 사용자 | 간단한 입력, 쉬운 목록 확인, 기본적인 피드백 |
| 하루에 먹은 음식을 간단히 관리하고 싶은 사용자 | 하루 동안 먹은 음식을 기록하고 나중에 확인하고 싶은 사용자 | 음식 히스토리 확인, 칼로리 참고, 기록 수정 및 삭제 |

FoodLog의 대표 사용자는 전문적인 식단 관리보다는 간단한 기록과 확인을 원하는 사용자로 설정한다. 따라서 화면과 기능은 복잡하지 않게 구성하고, 음식 기록의 추가, 조회, 수정, 삭제 흐름을 쉽게 이해할 수 있도록 한다.

## 3. 핵심 사용자 시나리오

### 3.1 음식 기록 추가 시나리오

사용자는 자신이 먹은 음식을 FoodLog에 새로 등록한다.

| 항목 | 내용 |
| --- | --- |
| 시작 조건 | 사용자가 음식 기록 목록 화면에 접속해 있다. |
| 사용자 목표 | 먹은 음식을 새 기록으로 저장한다. |
| 주요 입력값 | 음식 이름, 카테고리, 칼로리, 메모 |
| 기대 결과 | 새 음식 기록이 생성되고 목록에 표시된다. |

### 3.2 음식 기록 목록 조회 시나리오

사용자는 등록된 음식 기록을 목록에서 확인한다.

| 항목 | 내용 |
| --- | --- |
| 시작 조건 | 사용자가 FoodLog에 접속한다. |
| 사용자 목표 | 지금까지 기록한 음식 히스토리를 확인한다. |
| 표시 정보 | 음식 이름, 카테고리, 칼로리, 기록 일시 |
| 기대 결과 | 등록된 음식 기록이 목록 형태로 표시된다. |

### 3.3 음식 기록 상세 조회 시나리오

사용자는 목록에서 특정 음식 기록을 선택하여 상세 정보를 확인한다.

| 항목 | 내용 |
| --- | --- |
| 시작 조건 | 음식 기록 목록에 하나 이상의 기록이 있다. |
| 사용자 목표 | 선택한 음식 기록의 자세한 내용을 확인한다. |
| 표시 정보 | 고유 번호, 음식 이름, 카테고리, 칼로리, 메모, 기록 일시 |
| 기대 결과 | 선택한 음식 기록의 상세 정보가 표시된다. |

### 3.4 음식 기록 수정 시나리오

사용자는 잘못 입력한 음식 기록을 수정한다.

| 항목 | 내용 |
| --- | --- |
| 시작 조건 | 사용자가 음식 기록 상세/수정 화면에 접속해 있다. |
| 사용자 목표 | 기존 음식 기록의 내용을 수정한다. |
| 수정 가능 항목 | 음식 이름, 카테고리, 칼로리, 메모 |
| 기대 결과 | 수정된 내용이 저장되고 목록 또는 상세 화면에 반영된다. |

### 3.5 음식 기록 삭제 시나리오

사용자는 필요하지 않은 음식 기록을 삭제한다.

| 항목 | 내용 |
| --- | --- |
| 시작 조건 | 사용자가 음식 기록 상세/수정 화면에 접속해 있다. |
| 사용자 목표 | 불필요하거나 잘못 등록된 기록을 삭제한다. |
| 처리 대상 | 선택한 음식 기록 |
| 기대 결과 | 해당 음식 기록이 목록에서 제거된다. |

### 3.6 식단 피드백 확인 시나리오

사용자는 기록된 음식 데이터를 기준으로 간단한 식단 피드백을 확인한다.

| 항목 | 내용 |
| --- | --- |
| 시작 조건 | 음식 기록이 저장되어 있다. |
| 사용자 목표 | 자신의 식단 상태를 간단히 참고한다. |
| 기준 데이터 | 기록 개수, 총 칼로리, 카테고리 분포 |
| 기대 결과 | 규칙 기반 피드백 메시지가 표시된다. |

## 4. 사용자 여정 흐름

| 순서 | 단계 | 사용자 행동 | 시스템 역할 |
| --- | --- | --- | --- |
| 1 | 목록 화면 접속 | 사용자가 FoodLog에 접속한다. | 음식 기록 목록을 보여준다. |
| 2 | 음식 추가 | 사용자가 음식 추가 버튼을 누르고 정보를 입력한다. | 입력된 정보를 새 음식 기록으로 저장한다. |
| 3 | 기록 확인 | 사용자가 목록에서 음식 기록을 확인한다. | 저장된 음식 기록을 리스트로 표시한다. |
| 4 | 수정 또는 삭제 | 사용자가 특정 기록을 선택해 수정하거나 삭제한다. | 선택한 기록을 수정하거나 삭제한다. |
| 5 | 피드백 확인 | 사용자가 식단 피드백 화면으로 이동한다. | 기록 데이터를 기준으로 간단한 피드백을 제공한다. |

## 5. 시나리오별 상세 표

### 5.1 음식 기록 추가

| 단계 | 사용자 행동 | 시스템 반응 | 기대 결과 |
| --- | --- | --- | --- |
| 1 | 음식 기록 목록 화면에서 추가 버튼을 선택한다. | 음식 기록 추가 화면을 표시한다. | 사용자가 새 기록을 입력할 수 있다. |
| 2 | 음식 이름, 카테고리, 칼로리, 메모를 입력한다. | 입력값을 화면에 반영한다. | 등록할 음식 정보가 준비된다. |
| 3 | 등록 버튼을 선택한다. | 입력값을 확인하고 새 음식 기록을 생성한다. | 새 기록이 저장된다. |
| 4 | 목록 화면으로 이동한다. | 생성된 기록을 목록에 표시한다. | 사용자가 추가된 기록을 확인할 수 있다. |

### 5.2 음식 기록 목록 조회

| 단계 | 사용자 행동 | 시스템 반응 | 기대 결과 |
| --- | --- | --- | --- |
| 1 | FoodLog에 접속한다. | 저장된 음식 기록 목록을 불러온다. | 사용자가 기록한 음식 히스토리를 확인할 수 있다. |
| 2 | 목록 화면을 확인한다. | 음식 이름, 카테고리, 칼로리, 기록 일시를 표시한다. | 사용자가 음식 히스토리를 확인할 수 있다. |
| 3 | 특정 기록을 선택한다. | 선택한 기록의 상세 화면으로 이동한다. | 개별 음식 기록을 확인할 수 있다. |

### 5.3 음식 기록 상세 조회

| 단계 | 사용자 행동 | 시스템 반응 | 기대 결과 |
| --- | --- | --- | --- |
| 1 | 목록에서 특정 음식 기록을 선택한다. | 선택한 음식 기록의 상세 정보를 불러온다. | 사용자가 선택한 기록의 내용을 확인할 수 있다. |
| 2 | 상세 화면을 확인한다. | 음식 이름, 카테고리, 칼로리, 메모, 기록 일시를 표시한다. | 사용자가 상세 내용을 확인할 수 있다. |
| 3 | 뒤로가기 버튼을 선택한다. | 목록 화면으로 이동한다. | 사용자가 다시 전체 기록을 확인할 수 있다. |

### 5.4 음식 기록 수정

| 단계 | 사용자 행동 | 시스템 반응 | 기대 결과 |
| --- | --- | --- | --- |
| 1 | 상세/수정 화면에서 수정할 값을 입력한다. | 변경된 입력값을 화면에 반영한다. | 수정할 내용이 준비된다. |
| 2 | 저장 버튼을 선택한다. | 선택한 음식 기록의 수정 내용을 저장한다. | 음식 기록이 수정된다. |
| 3 | 수정된 내용을 확인한다. | 수정된 데이터를 화면에 표시한다. | 사용자가 수정 결과를 확인할 수 있다. |

### 5.5 음식 기록 삭제

| 단계 | 사용자 행동 | 시스템 반응 | 기대 결과 |
| --- | --- | --- | --- |
| 1 | 상세/수정 화면에서 삭제 버튼을 선택한다. | 삭제 요청을 처리한다. | 선택한 기록이 삭제 대상이 된다. |
| 2 | 시스템이 선택한 음식 기록을 삭제한다. | 해당 음식 기록을 목록에서 제거한다. | 삭제된 기록이 더 이상 표시되지 않는다. |
| 3 | 목록 화면으로 이동한다. | 삭제된 기록을 제외한 목록을 표시한다. | 사용자가 삭제 결과를 확인할 수 있다. |

### 5.6 식단 피드백 확인

| 단계 | 사용자 행동 | 시스템 반응 | 기대 결과 |
| --- | --- | --- | --- |
| 1 | 목록 화면에서 피드백 화면으로 이동한다. | 음식 기록 요약 정보를 표시한다. | 사용자가 피드백 화면을 확인할 수 있다. |
| 2 | 사용자가 화면의 요약 정보를 확인한다. | 기록 개수, 총 칼로리, 카테고리 요약을 표시한다. | 사용자가 음식 기록 현황을 확인할 수 있다. |
| 3 | 사용자가 피드백 메시지를 확인한다. | 기록 데이터를 기준으로 적절한 피드백을 제공한다. | 사용자가 자신의 식단 상태를 간단히 참고할 수 있다. |

## 6. 예외 상황

| 예외 상황 | 사용자 행동 | 시스템 반응 | 기대 결과 |
| --- | --- | --- | --- |
| 입력값이 비어 있는 경우 | 음식 이름, 카테고리, 칼로리 중 일부를 입력하지 않는다. | 필수 입력값이 필요하다는 메시지를 표시한다. | 사용자가 빠진 값을 입력한 뒤 다시 등록할 수 있다. |
| 칼로리에 숫자가 아닌 값을 입력한 경우 | 칼로리 입력란에 문자나 잘못된 값을 입력한다. | 숫자 입력이 필요하다는 안내를 표시한다. | 올바른 숫자 값으로 수정하도록 유도한다. |
| 삭제하려는 기록이 없는 경우 | 이미 삭제되었거나 존재하지 않는 기록을 삭제하려고 한다. | 기록을 찾을 수 없다는 메시지를 표시한다. | 사용자가 현재 목록 기준으로 다시 확인할 수 있다. |

## 7. 한 줄 요약

FoodLog는 사용자가 먹은 음식을 간단히 기록하고, 기록된 히스토리를 확인하며, 기본적인 식단 피드백까지 확인할 수 있는 음식 기록 서비스이다.
