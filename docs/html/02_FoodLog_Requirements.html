3. 기능 요구사항

3.1 음식 기록 추가

항목내용
기능 설명사용자가 먹은 음식을 새로 등록한다.
입력 항목음식 이름, 카테고리, 칼로리, 메모
처리 내용입력된 정보를 바탕으로 새 음식 기록을 생성한다.
결과생성된 음식 기록이 목록에 추가된다.
우선순위높음

3.2 음식 기록 목록 조회

항목내용
기능 설명등록된 음식 기록 전체를 목록으로 조회한다.
표시 항목음식 이름, 카테고리, 칼로리, 기록 일시
처리 내용저장된 음식 기록 목록을 불러온다.
결과사용자가 음식 기록 히스토리를 확인할 수 있다.
우선순위높음

3.3 음식 기록 상세 조회

항목내용
기능 설명특정 음식 기록의 상세 정보를 조회한다.
표시 항목id, 음식 이름, 카테고리, 칼로리, 메모, 기록 일시
처리 내용선택한 음식 기록을 기준으로 상세 데이터를 조회한다.
결과사용자가 개별 음식 기록의 상세 내용을 확인할 수 있다.
우선순위높음

3.4 음식 기록 수정

항목내용
기능 설명기존 음식 기록의 내용을 수정한다.
수정 항목음식 이름, 카테고리, 칼로리, 메모
처리 내용선택한 음식 기록의 수정 내용을 저장한다.
결과수정된 음식 기록이 목록과 상세 화면에 반영된다.
우선순위높음

3.5 음식 기록 삭제

항목내용
기능 설명등록된 음식 기록을 삭제한다.
처리 내용선택한 음식 기록을 삭제한다.
결과삭제된 음식 기록은 목록에서 제거된다.
우선순위높음

3.6 식단 피드백 확인

항목내용
기능 설명기록된 음식 데이터를 기준으로 간단한 식단 피드백을 확인한다.
기준 데이터음식 기록 개수, 총 칼로리, 카테고리 분포
처리 방식MVP 단계에서는 규칙 기반 피드백을 제공한다.
결과사용자가 자신의 식단 상태를 간단히 참고할 수 있다.
우선순위보통

4. 비기능 요구사항

요구사항설명우선순위
사용하기 쉬운 화면사용자가 목록, 추가, 상세, 피드백 화면을 쉽게 이동할 수 있어야 한다.높음
간단한 입력 구조음식 기록 입력 항목은 필수 정보 중심으로 구성한다.높음
사용자가 이해하기 쉬운 기록 관리 흐름음식 기록의 추가, 조회, 수정, 삭제 흐름이 명확하게 드러나야 한다.높음
향후 AI API 확장 가능성현재는 규칙 기반 피드백으로 구성하되, 이후 AI API 연동으로 확장할 수 있도록 고려한다.보통
가벼운 프로젝트 구조MVP 범위에 맞게 복잡한 인증, DB, 배포 구조는 제외한다.보통

5. 화면 요구사항

화면화면 목적주요 구성사용자 행동
음식 기록 목록 화면등록한 음식 기록 전체 확인음식 목록, 음식 요약 정보, 추가 버튼, 피드백 이동 버튼음식 추가, 상세/수정, 피드백 화면 이동
음식 기록 추가 화면새 음식 기록 입력 및 저장음식 이름, 카테고리, 칼로리, 메모 입력, 등록 버튼, 취소 버튼입력 후 등록하거나 취소
음식 기록 상세/수정 화면선택한 음식 기록 확인, 수정, 삭제상세 정보, 수정 입력 폼, 저장 버튼, 삭제 버튼, 뒤로가기 버튼내용 수정 또는 삭제 후 목록 이동
식단 피드백 화면음식 데이터를 바탕으로 피드백 확인총 기록 수, 총 칼로리, 카테고리 요약, 피드백 메시지피드백 확인 후 목록 이동

6. MVP 범위

기능포함 여부설명
음식 기록 추가포함사용자가 음식 기록을 새로 등록한다.
음식 기록 목록 조회포함전체 음식 기록을 목록으로 확인한다.
음식 기록 상세 조회포함특정 음식 기록의 상세 정보를 확인한다.
음식 기록 수정포함기존 음식 기록을 수정한다.
음식 기록 삭제포함불필요한 음식 기록을 삭제한다.
식단 피드백 확인포함규칙 기반으로 간단한 피드백을 제공한다.
주요 화면 설계포함목록, 추가, 상세/수정, 피드백 화면을 설계한다.

7. 제외 범위

제외 항목설명
로그인/회원가입MVP 단계에서는 사용자 인증 기능을 구현하지 않는다.
데이터베이스 저장현재는 DB 없이 서버 메모리의 임시 배열로 데이터를 관리한다.
실제 AI API 연동AI API는 향후 확장 방향으로만 정의하고 MVP에는 포함하지 않는다.
이미지 업로드음식 사진 등록 기능은 MVP 범위에서 제외한다.
복잡한 통계 기능일별, 주별, 월별 통계 기능은 향후 확장 기능으로 둔다.
