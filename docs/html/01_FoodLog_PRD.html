FoodLog PRD

FoodLog의 서비스 목적, 핵심 기능, MVP 범위, 화면 흐름, 향후 확장 방향을 정리한 제품 요구 문서입니다.

프로젝트명FoodLog
서비스 유형음식 기록 서비스
주요 기술Node.js, Express

PDF 저장 시 브라우저 인쇄 설정에서 머리글과 바닥글을 끄면 문서 본문만 깔끔하게 출력됩니다.

핵심 요약 FoodLog는 음식 기록 추가, 조회, 수정, 삭제 기능과 규칙 기반 식단 피드백을 중심으로 구성하는 MVP 서비스입니다. 초기 버전은 DB 없이 서버 메모리의 임시 배열을 사용합니다.

1. 프로젝트 개요

항목내용

프로젝트 단계프로젝트 초기 기획 및 화면 설계 단계

데이터 저장 방식DB 없이 서버 메모리의 임시 배열 사용

FoodLog는 사용자가 먹은 음식을 간단하게 기록하고, 기록한 음식 히스토리를 확인하며, 식단에 대한 기본적인 피드백을 받을 수 있는 음식 기록 서비스이다.

현재 단계에서는 복잡한 회원 기능이나 데이터베이스 연동 없이, 음식 기록의 추가, 조회, 수정, 삭제 기능과 간단한 식단 피드백 흐름을 중심으로 MVP를 구성한다.

2. 서비스 기획 배경

일상에서 먹은 음식을 꾸준히 기억하고 관리하는 것은 쉽지 않다. 특히 식단 관리를 처음 시작하는 사용자는 어떤 음식을 먹었는지, 하루 동안 어느 정도의 칼로리를 섭취했는지, 식단이 한쪽으로 치우치지는 않았는지 확인하기 어렵다.

FoodLog는 이러한 문제를 해결하기 위해 사용자가 먹은 음식을 빠르게 기록하고, 기록된 데이터를 기반으로 간단한 피드백을 제공하는 것을 목표로 한다.

3. 문제 정의

문제설명
음식 기록의 번거로움사용자가 매번 복잡한 입력을 해야 하면 기록을 지속하기 어렵다.
식단 히스토리 확인 어려움별도로 기록하지 않으면 과거에 무엇을 먹었는지 확인하기 어렵다.
간단한 식단 판단 기준 부족사용자는 자신의 식단이 적절한지 판단하기 어려울 수 있다.
프로젝트 범위 관리 필요초기 버전에서는 핵심 기능 중심으로 범위를 정리해야 한다.

4. 타깃 사용자

사용자 유형설명
식단 기록을 처음 시작하는 사용자복잡한 기능보다 간단한 음식 기록 기능이 필요한 사용자
하루 식사 내용을 확인하고 싶은 사용자자신이 먹은 음식 목록과 칼로리를 간단히 확인하려는 사용자
개인 식단 기록을 간단히 관리하고 싶은 사용자복잡한 관리 기능보다 기본 기록과 확인 기능을 원하는 사용자

5. 서비스 목표

  1. 사용자가 먹은 음식을 쉽고 빠르게 기록할 수 있도록 한다.
  2. 기록된 음식 목록을 통해 식단 히스토리를 확인할 수 있도록 한다.
  3. 음식 기록을 상세 조회하고 필요할 때 수정 또는 삭제할 수 있도록 한다.
  4. 기록된 음식 데이터를 기준으로 간단한 식단 피드백을 제공한다.
  5. 핵심 기능 중심으로 이해하기 쉬운 구조와 화면 흐름을 구성한다.

6.6 식단 피드백

기록된 음식 데이터를 기준으로 사용자에게 간단한 피드백을 제공하는 기능이다. 현재 MVP에서는 AI 분석이 아닌 규칙 기반 피드백으로 구성한다.

조건피드백 예시
총 칼로리가 높은 경우오늘 섭취 칼로리가 높은 편이므로 가벼운 식사를 고려한다.
특정 카테고리에 기록이 집중된 경우식단이 한 종류에 치우칠 수 있으므로 다양한 음식을 섭취한다.
기록된 음식이 적은 경우아직 기록이 적으므로 식사 후 꾸준히 기록한다.

7. 사용자 시나리오

단계사용자 행동시스템 동작
1사용자가 음식 기록 목록 화면에 접속한다.저장된 음식 기록 목록을 보여준다.
2사용자가 음식 추가 버튼을 선택한다.음식 기록 추가 화면으로 이동한다.
3사용자가 음식 이름, 카테고리, 칼로리, 메모를 입력한다.입력값을 받아 새 음식 기록을 생성한다.
4사용자가 등록을 완료한다.목록 화면에 새 음식 기록을 반영한다.
5사용자가 특정 음식 기록을 선택한다.상세 화면에서 음식 정보를 보여준다.
6사용자가 내용을 수정하거나 삭제한다.수정된 내용을 저장하거나 해당 기록을 삭제한다.
7사용자가 식단 피드백 화면을 확인한다.기록된 음식 데이터를 기준으로 간단한 피드백을 보여준다.

10. 향후 확장 방향

현재 FoodLog는 규칙 기반 피드백을 사용한다. 향후에는 AI API를 연동하여 개인화된 식단 피드백 기능으로 확장할 수 있다.

확장 기능설명
데이터베이스 연동음식 기록을 서버 재시작 후에도 유지
회원 기능사용자별 음식 기록 관리
날짜별 필터특정 날짜의 음식 기록만 조회
통계 기능일별, 주별, 카테고리별 칼로리 통계 제공
AI 식단 피드백AI API를 활용한 개인화 피드백 제공
이미지 업로드음식 사진을 함께 기록

11. 기대 효과

기대 효과 FoodLog를 통해 사용자는 자신이 먹은 음식을 간단히 기록하고, 기록된 히스토리를 통해 식단을 되돌아볼 수 있다. 기능 범위를 핵심 중심으로 유지하기 때문에 전체 서비스 흐름을 파악하기 쉽고, 이후 데이터베이스나 AI API 연동으로 확장하기에도 적합하다.
